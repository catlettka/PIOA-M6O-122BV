\## Файл `memory.py`



Файл `memory.py` содержит всю логику базы данных.



\### Реализовано:



\#### Структура базы данных

\- База данных реализована на основе `dict`

\- Поддержка нескольких таблиц

\- Возможность переключения между таблицами



\#### Работа с таблицами:

\- `create\_table` — создание новой таблицы

\- `use\_table` — выбор активной таблицы

\- `list\_tables` — список всех таблиц

\- `get\_table` — получение текущей таблицы



\#### CRUD операции:



\- \*\*CREATE\*\*

&#x20; - `create\_record` — добавление записи



\- \*\*READ\*\*

&#x20; - `select\_record` — выборка записей

&#x20; - фильтрация по полям



\- \*\*UPDATE\*\*

&#x20; - `update\_record` — обновление записей



\- \*\*DELETE\*\*

&#x20; - `delete\_record` — удаление записей



\#### Дополнительно:

\- сортировка записей по выбранному полю:

&#x20; - по возрастанию (ASC)

&#x20; - по убыванию (DESC)



\## Файл `tui.py`



Файл `tui.py` реализует консольный интерфейс пользователя (TUI).



\### Функции:

\- вывод меню

\- ввод данных пользователя (`input`)

\- обработка ошибок через `errors.py`

\- вызов функций из `memory.py`



\### Поддерживаемые команды:



\- 1. Создать таблицу

\- 2. Добавить запись

\- 3. Показать записи

\- 4. Обновить запись

\- 5. Удалить запись

\- 6. Список таблиц

\- 7. Переключить таблицу

\- 8. Сортировка записей

\- 0. Выход



\### Особенности интерфейса:

\- отображение текущей активной таблицы

\- проверка типов данных при вводе

\- защита от пустых значений

\- обработка ошибок сразу при вводе данных



\## Файл `\_\_main\_\_.py`



Точка входа в программу.



\- запускает интерфейс `TUI`

\- вызывает метод `run()`





\# Описание функциональности проекта



\## Работа с таблицами



\- создание новой таблицы

\- выбор активной таблицы

\- просмотр списка таблиц

\- переключение между таблицами



\## CRUD операции



\### CREATE

\- добавление новой записи

\- проверка типов данных

\- запрет пустых значений



\### READ

\- вывод всех записей

\- поиск и фильтрация по полям

\- сообщение при пустой таблице



\### UPDATE

\- обновление записей по фильтру

\- изменение отдельных полей

\- проверка корректности типов



\### DELETE

\- удаление записей по фильтру

\- возможность удаления всех записей (с подтверждением)



\## Дополнительно



\- сортировка записей:

&#x20; - по возрастанию

&#x20; - по убыванию

\- выбор поля сортировки пользователем
